# Security Vulnerability Remediation Plan

## Context

GitHub Dependabot found 50 vulnerabilities (2 critical, 18 high, 21 moderate, 9 low) in the portfolio project. The root cause is that the project is pinned to Next.js 13.4.19 (released mid-2023), which has multiple known CVEs, and many transitive dependencies are similarly outdated. Fixing this will bring the project up to a secure baseline and resolve the majority of vulnerabilities.

---

## Step-by-Step Process

### Step 1: Review Dependabot alerts on GitHub

Before making changes, get a full picture of what's flagged:

1. Visit: `https://github.com/mandyh101/portfolio-2025/security/dependabot`
2. Note which packages are flagged — most will trace back to `next` 13.x

---

### Step 2: Upgrade Next.js (fixes the most critical issues)

The current version is **Next.js 13.4.19**. The latest stable is **Next.js 15.x**. Upgrading to Next.js 14 (LTS) is the safest incremental step, as the project already uses the App Router.

**File to edit:** `package.json`

Change:
```json
"next": "13.4.19"
"eslint-config-next": "13.4.19"
"@next/mdx": "^13.4.19"
```
To:
```json
"next": "^14.2.29"
"eslint-config-next": "^14.2.29"
"@next/mdx": "^14.1.4"
```

Also update TypeScript type packages while here:
- `@types/node`: `20.5.3` → `^20.17.0`
- `@types/react`: `18.2.21` → `^18.3.0`
- `@types/react-dom`: `18.2.7` → `^18.3.0`
- `tailwindcss`: `3.3.3` → `^3.4.0`
- `postcss`: `8.4.28` → `^8.4.47`
- `eslint`: `8.47.0` → `^8.57.0`

Then run:
```bash
npm install
```

---

### Step 3: Run npm audit fix

After installing the updated packages:

```bash
npm audit fix
```

This will auto-patch remaining transitive dependency vulnerabilities where non-breaking fixes are available.

---

### Step 4: Handle remaining vulnerabilities

After `npm audit fix`, run `npm audit` again to see what's left. For anything still flagged:

**Option A — Force fix (may cause breakage, test after):**
```bash
npm audit fix --force
```

**Option B — Replace `@mapbox/rehype-prism`**
This package pulls in a chain of vulnerable build-time dependencies (mrm, libnpx). Since this is a syntax highlighter for blog posts, it can be replaced with `rehype-prism-plus` which has a cleaner dependency tree:
```bash
npm uninstall @mapbox/rehype-prism
npm install rehype-prism-plus
```
Then update the MDX config (likely in `next.config.js`) to use `rehype-prism-plus` instead.

---

### Step 5: Verify the app still works

```bash
npm run dev
```

Check:
- Homepage loads
- Projects section renders
- Blog pages render (if any are active)
- No console errors related to Next.js or MDX

Then do a production build check:
```bash
npm run build
```

---

### Step 6: Deploy and confirm Dependabot alerts are cleared

```bash
git add package.json package-lock.json
git commit -m "fix: upgrade Next.js to v14 and resolve security vulnerabilities"
git push
```

After pushing, revisit `https://github.com/mandyh101/portfolio-2025/security/dependabot` — most alerts should be resolved within a few minutes.

---

## Critical Files

- `package.json` — dependency versions to update
- `package-lock.json` — regenerated by `npm install`
- `next.config.js` — may need plugin import update if replacing `@mapbox/rehype-prism`

---

## Residual / Unfixable Issues

Some vulnerabilities have no upstream fix available:
- `cross-spawn` ReDoS (build-time only, no runtime risk)
- `parse-git-config` Prototype Pollution (build-time tool)
- `prismjs` DOM Clobbering (only affects syntax highlighting if user can inject HTML)

These are acceptable risks for a static portfolio site. GitHub will continue to flag them until upstream maintainers release patches.
